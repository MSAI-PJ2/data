## 3. 앱 개발 시 유의사항

### 중복 데이터 처리
- **기초 자살예방센터**와 **기초 정신건강복지센터**는 동일 주소·전화번호인 경우 다수 존재  
  (예: 강릉시, 원주시, 가평군 등)  
  → UI에서 같은 위치의 두 유형을 묶어 표시하거나 중복 추천을 방지하는 로직 검토 필요

### 재단 예외 처리
- **한국생명존중희망재단**은 서울 1곳뿐이므로 거리 기반 추천에서 별도 처리 권장  
  → 항상 하단 고정 표시 또는 추천 목록에서 제외

### 위기 상황 분기 로직 권장
```
사용자 상태 입력
    ↓
[즉각 위험 상황?]
    YES → 생명사랑위기대응센터 최우선 표시 + 109 / 119 안내
    NO  → 거리 기반 유형별 1곳씩 추천
```

### 운영시간 기반 필터링
- 야간(18:00 이후) / 주말·공휴일 접속 시  
  → **생명사랑위기대응센터** 및 **광역 센터 전화상담**만 활성 표시 권장  
  → 기초 센터는 "현재 운영 중이 아닐 수 있습니다" 안내 추가
